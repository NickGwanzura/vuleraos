# Comprehensive Build Plan: Items 1–5

Five independent features to complete the remaining gaps. Implementation order optimised to minimise rework.

---

## 1. Print & Document Formatting (easiest, highest impact)

What: Customisable PDF invoice/PO generation with business letterhead.

### Implementation
- **jsPDF** + `html2canvas` for client-side PDF generation (no server needed)
- Use the existing `DocumentTemplate` model to store letterhead config (logo URL, header text, colours)
- "Print" button on invoice and PO detail pages
- Customisable via `/settings/documents` — letterhead, layout, logo upload

### Files
```
npm install jspdf html2canvas
NEW: src/app/(dashboard)/settings/documents/page.tsx
NEW: src/app/(dashboard)/settings/documents/client.tsx
NEW: src/lib/print/index.ts — PDF generation helpers
NEW: src/components/layout/document-print.tsx — print-specific CSS
MOD: src/app/(dashboard)/sales/invoices/[id]/client.tsx — wire up print
MOD: src/app/(dashboard)/buying/purchase-orders/[id]/client.tsx — wire up print
NEW: src/app/api/settings/documents/route.ts — GET/PUT document template
```

## 3. Inflation-Adjusted Reporting

What: Financial reports that account for currency movement between transaction date and report date.

### Implementation
- `/reports/inflation-adjusted` page
- Takes a transaction amount + original date + exchange rate at that date
- Restates the value in current-day USD/ZWG using the exchange rate history
- Applies to: P&L summary, stock valuation report

### Files
```
NEW: src/app/(dashboard)/reports/inflation-adjusted/page.tsx
NEW: src/app/(dashboard)/reports/inflation-adjusted/client.tsx
NEW: src/app/api/reports/inflation-adjusted/route.ts
MOD: src/lib/currency/index.ts — add inflation adjustment helpers
```

## 4. Offline Sync Layer (PouchDB/CouchDB)

What: Basic offline-first capability — queue transactions locally, sync when online.

Note: A full CouchDB setup requires running CouchDB server. For now, I'll build the **client-side PouchDB layer** that:
- Stores pending transactions (offline-created invoices, stock movements) in IndexedDB via PouchDB
- Shows sync status indicator in the header
- Queues transactions when offline, syncs when connectivity returns

### Implementation
- Lightweight: detect online/offline status, queue failed API calls
- Basic PouchDB database per tenant for local storage
- Sync status dot in header (green/yellow/red)

### Files
```
npm install pouchdb-browser
NEW: src/lib/offline/index.ts — PouchDB setup, sync manager
NEW: src/lib/offline/queue.ts — offline transaction queue
MOD: src/components/layout/header.tsx — add sync status indicator
```

# Redis Integration Plan

## What
Wire the Redis URL into VuleraOS for caching, rate limiting, session management, and offline queue processing.

## Implementation

### 1. Install ioredis
```bash
npm install ioredis
```

### 2. Add Redis to `.env`
```
REDIS_URL="redis://default:auhebzrm4mf2punv@vuleraos-vuleradb-d8qwsz:6379"
```

### 3. Create Redis client (`src/lib/redis/client.ts`)
- Singleton client using ioredis
- Ping health check
- Graceful error handling (doesn't crash if Redis is down)

### 4. Caching Layer (`src/lib/redis/cache.ts`)
- `getCached(key, fetchFn, ttl)` — generic cache-aside pattern
- Used for: dashboard metrics, item lists, exchange rates
- TTL defaults: 60s for dynamic data, 300s for reference data

### 5. Rate Limiting (`src/lib/redis/rate-limit.ts`)
- Token bucket per API route per tenant
- Prevents API abuse

### 6. Offline Queue Processing (`src/lib/redis/queue.ts`)
- Replace localStorage queue with Redis list
- More reliable, persists across browser clears
- Can be processed server-side

### 7. Apply Caching to High-Traffic Endpoints
- `/api/stock/items` — cache item list (60s)
- `/api/sales/invoices` — cache invoice list (60s)
- `/api/settings/currency/rates/latest` — cache latest rate (300s)
- Dashboard page — cache metrics (30s)
